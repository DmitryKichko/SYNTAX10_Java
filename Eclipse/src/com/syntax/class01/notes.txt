

Что такое JRE, JDK, JVM?

Включает в себя:

- JVM (сам механизм запуска)
- Библиотека Java
- Системные файлы

JRE = JVM + библиотеки + средства для запуска приложения

!!! Если используем JRE, то можем запускать JAVA - программы, но не можем писать и компилировать их !!!

2) JDK (Java Development Kit) - Инструменты разработчика

Включает в себя:

- Всё, что нужно для написания и запуска Java - программ
- Компилятор javac (переводит .java -> .class)
- Отладчики, утилиты, документация

!!! JDK - используется для написания и запуска Java - программ

3) JVM (Java Virtual Machine) - Виртуальная машина/Мозг Java

Что делает:
- Читает байт-код (файлы: .class)
- Преобразует байт-код в машинные инструкции, понятные компьютеру
- Следит за безопасностью, памятью и управляет потоками

!!! JVM - это виртуальная машина, которая выполняет Java - программы. Как интерпретатор, который понимает язык
Java-программы, и общается с компьютером !!!

------------------------------------------------------------------------------------------------------------------------

Что такое компиляция кода и выполнение? Их порядок?

1) Компиляция - преобразование Java-кода (файл .java) в байт-код (файл .class), понятным для JVM (виртуальная машина).
Перевод кода на язык Java-машины

Пример: DataTypes.java --> DataTypes.class

2) Выполнение - это процесс, когда JVM читает байт-код и запускает программу шаг за шагом.

Что происходит первым при запуске программы:

Сначала происходит компиляция (.java --> .class), затем JVM выполняет байт-код из .class

Сначала переводит --> потом запускаем

------------------------------------------------------------------------------------------------------------------------

Запуск программы:

Компьютер понимает только 0 и 1 (машинный код)

Java - язык высокого уровня, понятный человеку

1) Пишем код на Java - исходный код на понятном языке

2) Компиляция - переводит .java файл в байт-код (Специальный код, понятный JVM, но не железу)

байт-код - не путать с машинным кодом, это промежуточный код

3) Происходит запуск, JVM загружает байт-код (.class) --> Проверяет безопасность (нет ли вредного кода) -->

--> Превращает байт-код в машинный код (Сначала интерпретирует (читает строчка за строчкой) --> при повторении переводит
части байт-кода в настоящий машиный код(0 и 1) --> ускоряет повторный запуск) --> Выполняет машинный код
на нашем процессоре
